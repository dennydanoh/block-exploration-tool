# 创意街区探索工具一期测试用例

## 1. 文档说明

本测试用例基于当前已确认的一期功能范围编写，覆盖以下模块：
- 首页 / 发现页
- 搜索结果页
- 地点详情页
- 登录 / 注册
- 收藏功能
- 个人中心
- 知识汇总台
- 全局导航与“无死胡同”检查

本文档的目标不是只验证功能是否可点击，而是验证用户在任意关键节点都不会进入无法继续、无法返回、无法恢复的死胡同。

## 2. 测试范围

### 2.1 一期范围内功能
- 浏览街区内容
- 搜索街区
- 查看地点详情
- 查看图片、店铺、外部链接、收藏数
- 用户注册、登录、退出
- 收藏/取消收藏街区
- 个人中心查看收藏街区
- 知识卡片编辑与发布
- 来源链接与图片来源管理

### 2.2 不在本轮重点范围
- 第三方真实接口联调
- 地图真实定位精度
- 推荐算法效果优劣
- 推送消息
- 第三方登录

## 3. 测试数据准备

### 3.1 测试账号
- `user_a`：已注册、无收藏
- `user_b`：已注册、已有 3 个收藏
- `user_c`：已注册、收藏已满多条，用于分页或长列表检查

### 3.2 测试地点
- `place_1`：南头古城，已上线，收藏数大于 0，来源完整
- `place_2`：鼓楼片区，已上线，收藏数为 0
- `place_3`：武康路安福路，已上线，来源完整，图片较多
- `place_4`：待发布街区，仅后台可见

### 3.3 测试来源
- 权威网站来源 1 条以上
- 小红书公开链接 1 条以上
- 大众点评公开链接 1 条以上
- 图片来源至少 2 条，带平台和用户名称

## 4. 通过标准

### 4.1 功能通过标准
- 核心流程全部可完成
- 关键状态切换正确
- 数据展示正确
- 错误提示清晰

### 4.2 无死胡同标准
- 每个页面都能回首页、上一层或进入可继续操作的页面
- 未登录触发收藏时，必须给出登录入口
- 空状态页面必须有明确下一步动作
- 失败状态必须有重试、返回或替代路径

## 5. 测试用例

## 5.1 全局导航与无死胡同

### TC-NAV-001 首页全局导航可达
前置条件：
- 站点正常打开

步骤：
1. 进入首页
2. 查看顶部导航
3. 依次点击 `发现`、`登录`、`个人中心`

- 点击后进入对应页面

无死胡同检查：
- 每个目标页面都能继续返回首页或跳转到其他主页面

### TC-NAV-002 搜索页全局导航可达
前置条件：
- 已进入搜索结果页

步骤：

2. 点击 `发现`
3. 再回到搜索页
4. 点击 `登录` 和 `我的收藏`

预期结果：
- 所有入口正常跳转
- 返回搜索页后筛选条件不丢失或有合理保留策略

无死胡同检查：
- 搜索页不能成为只能后退浏览器才能离开的页面

### TC-NAV-003 地点详情页返回路径完整
前置条件：

无死胡同检查：
- 详情页至少存在两条离开路径：返回主入口、跳转个人中心或其他主页面

### TC-NAV-004 登录后导航状态切换

前置条件：
- 游客态

步骤：
1. 点击 `个人中心`

## 6. 回归优先级

### P0 必测
- 首页打开
- 搜索可用
- 地点详情可用
- 登录成功
- 收藏成功
- 取消收藏成功
- 个人中心展示收藏
- 全站收藏数展示正确
- 外部链接可点击
- 后台知识卡片可发布

### P1 重点回归
- 无结果页
- 空收藏页
- 图片来源展示
- 来源链接管理
- 异常恢复

### P2 补充回归
- 长列表
- 数字展示格式
- 弱网表现

## 7. 建议执行顺序

1. 先跑 P0 主流程
2. 再跑登录与收藏链路
3. 再跑个人中心和收藏数同步
4. 再跑知识汇总台
5. 最后跑异常与无死胡同检查
